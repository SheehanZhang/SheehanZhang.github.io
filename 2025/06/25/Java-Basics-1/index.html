Java Basics 1

The Java Basics X series are study notes for MIT 6.092, specifically for basic java syntax.

Basic Knowledge:

  • Java runs on a virtual machine, Java Virtual Machine (JVM).

  • Java is more complex than Python while simpler than C++.

  • The compiling process:
    Source Code (.java) -> javac -> Byte Code (.class) -> java

Types: primitive and reference.

  • Primitive type: basic java types, e.g. int, long, stored in the variable (stack)

  • Reference type: arrays and objects, stored the address of the content in variable (stack), the real content is stored in heap.

An array must be given a specific size:
int[] name = new int[5];

Curly braces can be used to initialize an array:
int[] values = { 12, 24, -23, 47};
int size = values.length size=4

Different types have different operators rules:
double a = 2/3; a=0.0
int c = 5/2; c=2, both are integers, integer division will cut the decimal part.
double c = 5.0/2.0; c=2.5, both 5.0 and 2.0 are double, floating point division.
double c = 5/2.0; c=2.5
double c = 5.0/2; c=2.5, one number is a double, so Java automatically converts the other to double.
double c = 5/2; c=2, 5/2 is an integer division, the reuslt is 2, then 2 is assigned to c, which is a double, thus the result is 2.0.
double a = (doulbe)2/3; a=0.6666..., integer 2 is converted to double 2.0, then 2.0/3 is double division

Variable Scope: variables live in the block ({}) where they are defined (scope).

Operators:

  • Assignment: =
  • Addition: +
  • Subtraction: -
  • Multiplication: *
  • Division: /
  • concatenation: string + string

while & for:

break termates a for or while loop
continue skips the current iteration of a loop and proceeds directly to the next iteration

class

The object’s location is called a reference.
If comparing two instances of the same class, it is comparing their addresses.
When MyClasss obj = new MyClass();, obj is a reference, itself is in stack, all the fields of the obj is located in heap.

static type and method

defined for the class declaration, not unique for each instance.

  • the static fields belong to the class, not belong to specific instances

What can be static?

  • static variable/field, belongs to class not instance, access by ClassName.FieldName
  • static method, belongs to class not instance, cannot access instance field/methods
  • static block, execute once when the class is loaded, can only access static member
  • static nested class, donot need to create outer class instance, cannot access outer class non-static member
  • Non-static methods can reference static methods, but not the other way around, why?
  1. static member belongs to class, non-static belongs to instance/object.
  2. access static member: class.staticMember; access non-static member: object.non-staticMember
  3. memory allocation: when a class is loaded, static is in method while the non-static member is in heap when the object is created. A static member is existed when the class is loaded, even if no object of the class has been created. A non-static member is existed unless new an object.
  4. when a non-static member is exectued, it has a this pointer. Static member does not have. So non-static methods has context to access the static member while static does not.
  5. when the JVM initializes the class, the static methods is prepared (in methods area). Non-static member only exists when new an object.
  • Why is main methods static?
    When JVM runs, no object is created, so the main method must be executed by class itself, not object.

首先一个知识点事 primitive 和 reference 变量，基本类型就是常见的类型，int, long 等等，存储在 stack 中，stack 中直接就是存储真实数据；而引用类型有两种，数组和对象，在 stack 中存储的是指向堆的地址，堆中存放的是真正的数据。对于实例对象的字段，无论是基本类型还是引用类型，都放在堆中。静态字段在方法区/元空间(Metaspace)中。

对于精度，java 会自动靠齐高精度，也就是精度对齐，从小到大顺序是 byte, short/char, int, long, float, double。对于编程语言中的算式，要从等号右边开始看，等号的右边部分从左往后看，然后再看等号左边。 例如 double c = 5/2,首先是，5 和 2 的整数除法，去掉小数点部分，得到结果 2，然后赋值给变量 c，c 是一个 doulbe 类型，所以结果变成 2.0。这样一个思考流程需要牢记。

变量存活在{}中，if 中新声明的变量出了 if 就不能用了。

静态成员，有四种，静态字段，静态方法，静态段，静态内部类

静态字段，只属于 class，也就是不管你有实例 object 还是没有，不管你有多少个 object，静态字段只有一个，所有 object 共享

静态方法，同理，只属于 class，静态方法只能访问静态字段，但一般的（也就是实例方法）可以访问静态方法和静态类，但静态方法不能访问实例方法

静态段，当 class 执行的时候执行一次，用 static{}，里面只能访问静态成员

静态内部类，外部类.内部类=new 外部类.内部类()，可以不需要实例化外部类就可以实例化内部类，不能访问外部类的非静态成员
